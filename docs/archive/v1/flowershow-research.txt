⏺ Flowershow 功能概览

  Flowershow 是一个将 Markdown 文件转化为精美网站的工具，特别适合 Obsidian
  用户。核心功能如下：

  发布方式

  - Obsidian 插件 — 直接从 Obsidian vault 一键发布
  - GitHub 同步 — 连接 GitHub 仓库自动部署
  - CLI 命令行 — 本地文件夹直接发布，无需 Git

  内容格式支持

  - 标准 Markdown + MDX
  - Obsidian Canvas（.canvas 可视化图表）
  - Obsidian Bases（数据库视图，Beta）
  - LaTeX 数学公式（KaTeX）
  - Mermaid 流程图
  - 内嵌 HTML

  网站配置

  - 导航栏：Logo、链接、下拉菜单、CTA 按钮
  - 侧边栏 + 目录（TOC）
  - 页脚：链接 + 社交图标
  - 暗色/亮色/跟随系统 主题切换
  - 多款官方主题

  博客 & 内容管理

  - 博客系统（文章列表、作者页面）
  - RSS 订阅
  - 评论系统（Giscus + GitHub Discussions）
  - 表单（Newsletter、联系、问卷）
  - 站内搜索
  - 密码保护（付费功能）
  - URL 重定向

  SEO & 分析

  - Google Analytics 4 / Umami 分析
  - SEO 元数据、Open Graph、Twitter Card
  - 自定义域名（付费功能）

  自定义

  - 自定义 CSS、Google Fonts、Favicon
  - "Edit this page" 链接到 GitHub

  技术栈

  底层基于 Next.js / Vercel，支持自动同步工作流，分免费和付费两个层级。

  ---
  总结：Flowershow 本质上是一个 Obsidian / Markdown → 网站
  的发布平台，主打简单易用，适合做文档站、知识库、博客或 landing
  page。对标产品类似 Notion 的 Super.so 或 Obsidian Publish，但更开放（支持
  GitHub 同步和 CLI）。

  Flowershow 技术原理

  核心架构很清晰：

  1. 用户上传 Markdown 文件（通过 CLI / Obsidian 插件 / GitHub 同步）
  2. Cloudflare Worker 在边缘处理 Markdown → HTML 转换
  3. Next.js 应用负责渲染和路由
  4. 后端服务：PostgreSQL（元数据）、MinIO/S3（文件存储）、Inngest（任务队列）

  本质上就是：文件上传 → 存储 → 边缘转换 → 动态渲染。

  ---
  基于 Cloudflare 做一个精简版，完全可行

  你手上有 Cloudflare 全家桶，刚好能覆盖所有核心组件：

  ┌──────────────────────┬────────────────────────────────────┐
  │         需求         │          Cloudflare 方案           │
  ├──────────────────────┼────────────────────────────────────┤
  │ 文件存储             │ R2（存 Markdown 原文件）           │
  ├──────────────────────┼────────────────────────────────────┤
  │ 元数据/用户管理      │ D1（SQLite，存站点信息、文件索引） │
  ├──────────────────────┼────────────────────────────────────┤
  │ Markdown → HTML 转换 │ Worker（用 marked 或 markdown-it） │
  ├──────────────────────┼────────────────────────────────────┤
  │ 网站渲染/路由        │ Workers 或 Pages                   │
  ├──────────────────────┼────────────────────────────────────┤
  │ 文件上传 API         │ Worker 接收上传                    │
  ├──────────────────────┼────────────────────────────────────┤
  │ 自定义域名           │ Cloudflare DNS 原生支持            │
  ├──────────────────────┼────────────────────────────────────┤
  │ CDN/缓存             │ 自带                               │
  └──────────────────────┴────────────────────────────────────┘

  一个精简版 MVP 可以做到的功能

  核心功能（完全可实现）：
  - 用户注册/登录（D1 + JWT）
  - 上传 Markdown 文件/文件夹 → 自动生成网站
  - 多文件支持，自动生成侧边栏导航
  - 文件管理（增删改、重新上传）
  - {username}.yourdomain.com 子域名路由
  - 暗色/亮色主题
  - 基本 SEO（title、description、OG 标签）

  中等难度（可以做）：
  - 在线 Markdown 编辑器（Monaco / CodeMirror）
  - 文件目录树拖拽排序
  - 自定义 CSS/主题
  - 搜索功能（用 D1 全文检索或 Workers AI）
  - RSS 生成

  进阶（后期加）：
  - CLI 工具上传
  - GitHub 同步
  - Obsidian 插件
  - 自定义域名绑定
  - 密码保护页面

  大致架构

  用户浏览器
      │
      ▼
  Cloudflare Worker (路由层)
      │
      ├── /api/upload    → 接收文件 → 存入 R2 + 索引写入 D1
      ├── /api/sites     → 站点管理 CRUD
      ├── /api/auth      → 登录注册
      │
      └── /{user}/{site}/{path}  → 从 R2 取 Markdown
                                  → Worker 转 HTML
                                  → 注入模板（导航/主题/SEO）
                                  → 返回完整页面
                                  → KV 缓存热门页面

  - D1：5GB 免费
  - KV：每天 10 万次读免费

  小规模运营基本零成本。
