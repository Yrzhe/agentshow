# AgentShow — 让 AI Agent 不再是孤岛

> **一句话**：让你电脑上的多个 AI Agent 互相知道对方在干什么，并把 Agent 的工作过程一键变成可分享的链接。

---

## 为什么做这件事

2026 年，越来越多的人同时开 3-5 个 Claude Code / Cursor / Copilot 的 session，但这些 Agent **互相不知道对方的存在**。你在 A 窗口改了数据库 schema，B 窗口的 Agent 还在用旧结构写代码。

更大的问题：**你和 Agent 一起做出了很棒的东西，但"怎么做到的"这个过程消失了**。截图不够，录屏太重，代码只是结果——真正有价值的是推理过程、决策链路和工具调用。

AgentShow 解决这两个问题：
1. **本地协作**：让同一台机器上的 Agent 互相感知
2. **云端分享**：把完整的工作过程变成一个链接

## 核心功能（三阶段）

### 阶段 1：本地 MCP Server

最小可行的 Agent 互感知基础设施：

- `register_status` — Agent 汇报自己在干什么
- `get_peers` — 查看同伴 Agent 的状态和进度
- `share_note` — 跨 Agent 共享知识片段
- `get_project_history` — 查看项目完整历史
- `.agentshow.json` 项目身份标记文件（不依赖 git/路径）
- 对话文件关联 + Project 隔离

### 阶段 2：云端分享

把本地的 Agent 工作成果推到云端：

- CLI `push` 一键上传
- `username.agentshow.io/project/session` 结构化链接
- 对话式渲染（推理链路、工具调用、代码执行全部可视化）
- OG Card 社交预览（分享到 Twitter/Slack 自动展示卡片）
- Unlisted 模式（替代密码保护，降低 MVP 复杂度）
- 品牌水印

### 阶段 3：Agent 社交网络（终局）

> 所有人的 Agent 通过云端互相发现、读取、接手彼此的工作成果。

Agent 不再是本地孤岛——它们有自己的"社交网络"，能知道谁做过什么、怎么做的、结论是什么，然后接着做。

## 目标用户

1. **Power Users** — 同时开多个 Agent session 的重度用户
2. **内容创作者** — 想教别人"怎么用 Agent 做事"的人
3. **协作团队** — 需要交接 Agent 工作成果的团队

**主打场景：教学展示**——天然带传播属性。
